# Guion breve para video

Duración sugerida: 3 a 4 minutos.

## 1. Introducción

Nuestro proyecto aborda el problema de interpretar datos electorales cuando existen muchos registros, opciones y resultados. La solución no busca afirmar fraude ni reemplazar una revisión oficial, sino identificar señales atípicas en datos simulados y explicarlas de forma clara.

## 2. Objetivo

El objetivo fue construir una herramienta funcional que procese datos electorales simulados, calcule métricas, compare resultados con su contexto y genere un informe visual con interpretación neutral.

## 3. Metodología

Se usaron datos simulados en un entorno controlado. Esto permite validar el comportamiento de la herramienta y comprobar si identifica correctamente casos llamativos.

El sistema calcula métricas como participación, concentración de votos y comparación con mesas cercanas. También revisa calidad básica de datos y muestra qué información fue usada para interpretar el caso.

## 4. Solución

La aplicación funciona con un flujo guiado: primero procesa los datos, luego genera una lectura de datos del caso principal y finalmente crea un informe con gráficas y conclusiones neutrales.

El informe muestra tarjetas de métricas, señales principales, visualizaciones, comparación contextual y un anexo técnico.

## 5. Resultados

En el caso simulado, la mesa analizada tiene una participación del 98%, mientras que el promedio del puesto es 78.1%. Además, una opción concentra el 97.7% de los votos, frente a un promedio comparable de 58%.

Estos valores sugieren un comportamiento atípico frente al contexto disponible. Sin embargo, el sistema mantiene una interpretación neutral: la evidencia no permite afirmar fraude electoral ni atribuir responsabilidad.

## 6. Conclusión

El valor del proyecto está en transformar datos electorales simulados en una lectura visual, clara y justificable. La solución cumple el objetivo porque permite analizar, interpretar y comunicar señales atípicas de forma comprensible.
